What’s in a name? That which we call a rose By any other word would smell as sweet — William Shakespeare, Romeo and Juliet

I have plans for re-implementing several of the data structures I wrote about in Functional Data Structures in R. I want a package with useful data structures, and I did write one, named ralgo, when I wrote the data structure book, but many of the data structures can be implemented much cleaner if I use the pattern matching features of pmatch and, where possible, optimise the code using tailr.

I need a good name for such a package, though. I made a GitHub repository named pmtypes—for “pattern matching types”—but that is a boring name.

Please help me come up with a good name.
